# Phase 08.1: Update skills in SVGs based on resume without grimoire suffix - Research

**Researched:** 2026-04-12
**Domain:** SVG Generation, Content Update
**Confidence:** HIGH

## Summary

The current SVG programmatic generation uses a "Grimoire" theme (`grimoire-frontend-dark-only.svg`, title "Grimoire of Frontend"). Based on the requirement to remove the "grimoire" terminology and align skills with the resume, we must update the generation script to output simpler titles (e.g., "Frontend Engineering") and extract precise skill sets from `Thushar_Sreenivas_Resume.pdf`.

**Primary recommendation:** Update `scripts/generate-magic-circles.js` to rename output files and adjust internal SVG text paths, modify `README.template.md` to reference the new files, and update `verify-svgs.js` to assert the new filename pattern.

## User Constraints

- **Remove "grimoire" suffix**: Drop "Grimoire of X" from SVG text and rename `grimoire-*-dark-only.svg` file names.
- **Update Skills based on resume**: Extract exact technologies listed under the "Technical Skills" section of `Thushar_Sreenivas_Resume.pdf`.
- **Maintain Dark Theme SVGs**: The pipeline generates `-dark-only.svg` files to avoid light/dark mismatch issues.

## Architecture Patterns

### Pattern: Programmatic Content Generation
**What:** SVGs are generated by running `scripts/generate-magic-circles.js`, which then gets injected/validated by subsequent scripts.
**When to use:** Whenever underlying data (like skills) changes, you modify the generator, not the SVGs.
**Example:**
```javascript
['frontend', 'backend', 'tooling'].forEach(skill => {
  generateSkillBadge(skill, `skill-${skill}-dark-only.svg`);
});
```

## Common Pitfalls

### Pitfall 1: Broken Build Pipeline Validations
**What goes wrong:** `npm run build` fails with `Missing required grimoire-* SVG badge replacement files`.
**Why it happens:** `scripts/verify-svgs.js` has a hardcoded regex check for `grimoire-`.
**How to avoid:** Update `verify-svgs.js` to check for `skill-` or whatever the new prefix is.

### Pitfall 2: SVG Text Overflow
**What goes wrong:** The skills list is too long and gets clipped by the SVG `viewBox`.
**Why it happens:** SVGs do not auto-wrap text paths. The available width is ~520px.
**How to avoid:** Keep comma-separated skill strings under ~45 characters.
- *Frontend:* `React 18, TypeScript, Redux, Tailwind CSS` (42 chars)
- *Backend:* `Node.js, Go, PostgreSQL, AWS, Redis` (35 chars)
- *Tooling:* `GitHub Actions, Vite, Docker, Playwright` (40 chars)

### Pitfall 3: Orphaned Image Files
**What goes wrong:** Old `grimoire-*.svg` files remain in `assets/images/` and get pushed to Git.
**Why it happens:** `fs.writeFileSync` doesn't delete old unused files.
**How to avoid:** Specifically remove `assets/images/grimoire-*.svg` or instruct the planner to clean up the assets folder before generation.
