# Phase 5: Fix Template Architecture & Asset Integration

## Goal
Fix template architecture and asset integration gaps found in audit.

## Steps
1. **Update `README.template.md`**:
   - Change asset paths to correctly point to `assets/images/...` instead of `assets/...`.
   - Update file names to match what `generate-assets.js` produces (e.g., `hero-banner-light.svg`, `divider-light.svg`, `header-journey-light.svg`, etc.).
   - Refactor the `<img>` tags into `<picture>` tags so that `build-template.js` can process them as originally intended.
2. **Update `build-template.js`**:
   - Parse `<picture>` tags and properly output `<img>` tags with `#gh-light-mode-only` and `#gh-dark-mode-only` suffixes, retaining the `width` and `alt` attributes. This ensures the compiled `README.md` uses GitHub's proprietary mode-switching syntax while retaining formatting.
3. **Verify Pipeline**:
   - Run `npm run build`.
   - Ensure the generated `README.md` references the correct `assets/images/` paths and has the proper `<img>` tags.
